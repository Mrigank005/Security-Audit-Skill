━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 Location: src/routes/auth.js, Lines 13, 45

🔍 Vulnerable Code:
```js
const hashedPassword = crypto.createHash('md5').update(password).digest('hex');
```

⚠️ Risk:
  MD5 is cryptographically broken and unsuitable for password hashing. It is
  extremely fast to compute, enabling brute-force attacks. Rainbow tables for
  MD5 are widely available. An attacker who obtains the database can crack most
  passwords in minutes.

✅ Recommended Fix:
  Use bcrypt with a cost factor of at least 12:
```js
const bcrypt = require('bcrypt');

// Hashing
const hashedPassword = await bcrypt.hash(password, 12);

// Verifying
const isValid = await bcrypt.compare(password, storedHash);
```

🟡 MEDIUM — Weak Random Number Generation for Session Tokens
Confidence: HIGH
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 Location: src/utils/helpers.js, Lines 23–27

🔍 Vulnerable Code:
```js
function generateSessionToken() {
  const token = Math.random().toString(36).substring(2) +
                Math.random().toString(36).substring(2) +
                Date.now().toString(36);
  return token;
}
```

⚠️ Risk:
  `Math.random()` is not cryptographically secure. Its output is deterministic
  given the seed, and can be predicted by observing a few outputs. Combined with
  `Date.now()` (also predictable), an attacker can forge valid session tokens.

✅ Recommended Fix:
```js
const crypto = require('crypto');

function generateSessionToken() {
  return crypto.randomBytes(32).toString('hex');
}
```
